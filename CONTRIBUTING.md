### Reporting Bugs

Found a bug? ğŸ› Help us fix it! When reporting bugs:

1. **Check existing issues** to make sure it hasn't been reported
2. **Use a clear title** that describes the problem
3. **Provide detailed information**:
   - Clear description of the issue
   - Steps to reproduce the bug
   - Expected vs actual behavior
   - Screenshots or error messages (if applicable)
   - Your environment (browser, OS, Node version, etc.)

**Example:**
```
Title: [Bug] Navigation menu not showing on mobile devices

Description:
The navigation menu doesn't appear when viewing the site on mobile devices (iPhone 12, Safari).

Steps to Reproduce:
1. Open site on mobile device
2. Click hamburger menu
3. Menu doesn't appear

Expected: Menu should slide in from the side
Actual: Nothing happens

Environment:
- Device: iPhone 12
- Browser: Safari 15.0
- OS: iOS 15
```

### Suggesting Enhancements

Have an idea? ğŸ’¡ We'd love to hear it! When suggesting enhancements:

1. **Check existing issues** to see if it's been discussed
2. **Use a clear title** starting with `[Feature]` or `[Enhancement]`
3. **Explain your idea**:
   - What would this feature do?
   - Why would it be useful for learners?
   - How would it improve the learning experience?
   - Any examples or mockups?

**Example:**
```
Title: [Feature] Add dark mode toggle

Description:
Add a dark mode toggle to improve the reading experience, especially for users who prefer dark themes.

Why it matters:
Many developers prefer dark themes, and it reduces eye strain during long study sessions.

Proposed Solution:
- Add a toggle in the header
- Persist preference in localStorage
- Use Tailwind dark mode utilities
```

### Pull Requests

1. **Fork the repository**
   ```bash
   git clone https://github.com/khaleedbinaziz1/open-stack-js.git
   cd open-stack-js
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Install dependencies**
   ```bash
   npm install
   ```

4. **Make your changes**
   - Follow the existing code style
   - Add comments where necessary
   - Update documentation if needed

5. **Test your changes**
   ```bash
   npm run build
   npm run dev
   ```

6. **Commit your changes**
   ```bash
   git commit -m "Add: description of your changes"
   ```

7. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **Open a Pull Request**
   - Provide a clear title and description
   - Reference any related issues
   - Wait for review and feedback
